# تقييم حالة النظام الآن

من آخر السطور لديك:

```text
✓ Ready in 674ms
GET / 200
GET /api/projects 200
```

هذا يعني أن:

| النظام             | الحالة |
| ------------------ | ------ |
| Node               | يعمل   |
| Next.js server     | يعمل   |
| API projects       | يعمل   |
| filesystem reading | يعمل   |
| Dashboard          | يعمل   |

أنت الآن تشغل مشروع مبني على Next.js بنجاح.

الوثائق:
[https://nextjs.org/docs](https://nextjs.org/docs)

# المشكلة التي حدثت

ظهرت هذه الرسالة:

```text
Port 3000 is in use
Unable to acquire lock
```

السبب:

هناك **نسخة أخرى من السيرفر كانت تعمل**.

أنت حليتها بشكل صحيح باستخدام:

```powershell
taskkill /PID 33168 /F
```

وهذا بالضبط ما يفعله المطورون.

# الأوامر التي يجب أن تحفظها (الأهم)

## تشغيل المشروع

```powershell
cd C:\ALI_WORKSPACE\ali-dashboard
npm run dev
```

---

## إيقاف السيرفر

في نافذة السيرفر اضغط فقط:

```
CTRL + C
```

لا تكتبها.

---

## إذا كان المنفذ مستخدم

اعرف العملية:

```powershell
netstat -ano | findstr :3000
```

أوقفها:

```powershell
taskkill /PID NUMBER /F
```

# خطأ كنت تفعله (وتجنبه)

كنت تكتب أشياء مثل:

```powershell
fs.readdirSync()
security vulnerabilities
dependency issues
CTRL + C
```

هذه ليست أوامر PowerShell.

| الشيء                    | أين يستخدم        |
| ------------------------ | ----------------- |
| fs.readdirSync           | داخل الكود        |
| CTRL+C                   | ضغط لوحة المفاتيح |
| security vulnerabilities | نتيجة npm audit   |
